# 블로그 톤북 (Tone Book) v2

매일 발행되는 '12시에 만나요' 블로그 시리즈의 **단일 스타일 소스(SoT)**.
파이프라인 모든 에이전트(작성/이미지/SEO/품질)는 이 문서를 우선 참조한다.

페르소나: 모바일 네이버 앱으로 점심시간에 주식 정보 찾는 30~40대 직장인 (초보~중수).

**v2 핵심 변경 (2026-05-18)**: 트래픽 성장 위주로 SEO 키워드(§1, §11)·체류시간(§12)·발행 시점 분산(§13) 룰 신설.
브랜드 키워드("12시에 만나요")만으로는 검색 노출이 한정적이라, **종목명/시황 검색어를 본문·제목 앞쪽에 강제 배치**한다.

---

## 1. 제목 룰 (Title Rule) — v2 SEO 강화

### 형식
```
[종목/지수 키워드] [핵심 숫자/이벤트] — [후킹 문구]
```

### 규칙
- **공백 포함 28~32자**. 26자 미만은 정보 부족, 33자 이상은 모바일 검색 결과에서 잘림.
- **검색 수요 있는 메인 키워드는 앞 12자 이내에 무조건 포함** — 종목명(`SK하이닉스`, `삼성전자`, `LG에너지솔루션`) 또는 지수(`코스피`, `코스닥`) 또는 날짜형 시황 키워드(`5월 18일 시황`).
- **브랜드 키워드("12시에 만나요") 제목 노출 금지** — 검색 트래픽 0. 태그·시리즈 표시는 본문 부록에.
- **세컨더리 검색 키워드를 후킹 문구에 1개 더 포함** — 예: `매수 타이밍`, `목표가`, `매도 신호`, `반등`, `조정`, `전망`, `차트 분석`, `수급`.
- **`[2026.05.12]` 같은 날짜 접두어 금지**. 날짜는 본문 메타에 둔다.
- **물음표 1개까지만 허용**. 느낌표·이모지·괄호 다중 사용 금지.
- **숫자 콤마 포함**: `8,000`, `7,400`, `1.1조` 형식 통일.

### 제목 후보 3안 생성 룰 (v2)
에이전트 4는 제목을 1개만 뽑지 말고 **3안**을 생성해 키워드 매트릭스로 평가:
1. **종목 우선안** — 가장 강조된 종목명 + 핵심 액션 (예: `SK하이닉스 18만원 반등 — 외국인 매수 전환 신호`)
2. **지수/시황 우선안** — 코스피·코스닥 + 시황 키워드 (예: `코스피 6,500 안착 — 5월 18일 외국인 1.2조 매수`)
3. **이벤트 우선안** — 매크로/이벤트 + 종목 (예: `FOMC 직전 코스피 변동성 — 반도체 비중 줄여야 할 때`)

선택 기준: 영상 핵심 메시지에 가장 부합하는 안 + 검색량 큰 키워드. 점수 동률이면 종목 우선안 채택.

### Good vs Bad
- ❌ `[2026.05.12] 코스피 8천피 목전 반락 이유 — 외국인 1.1조 매도의 진짜 의미` (38자, 키워드 뒤)
- ❌ `12시에 만나요 5월 12일 — 코스피 반락 이유` (브랜드 키워드 노출, 검색 트래픽 0)
- ✅ `코스피 8,000 직전 반락 — 외국인 1.1조 매도 의미` (29자, 키워드 앞, 세컨더리 "외국인 매도")
- ❌ `빅테크 실적이 말해주는 반도체 다음 기대 업종은?` (메인 종목 모호)
- ✅ `반도체 다음 주도주, SK하이닉스 매도 이후 섹터 순환` (30자, 종목명 + "섹터 순환")

## 2. 첫 100자 후킹 공식

`# 제목` 직후 첫 문단은 **공백 포함 100~150자**, 다음 3요소를 포함:
1. **핵심 결론** (한 줄로 단정)
2. **핵심 키워드 2회 반복** (자연스럽게)
3. **숫자 1개** (관심 유발용)

### 템플릿
```
[핵심 결론 한 줄]. [숫자/이벤트] [핵심 키워드]가 [무엇을 의미하는지]. [독자에게 던지는 질문 한 줄].
```

### 예시
> 코스피가 8,000선 턱밑에서 꺾였습니다. 외국인이 하루 1.1조 원을 던진 이 코스피 반락, 단기 조정인지 추세 전환의 신호인지. 직장인 투자자가 오늘 확인해야 할 수급 데이터를 정리합니다.

> **금지**: 영상 정보 테이블(채널/출연자/타임코드)을 첫 100자에 두는 것. 이는 본문 최하단 부록으로.

## 3. 본문 구조 — v2 체류시간 강화

### 발행 순서 (위에서 아래)
```
1) # 제목
2) [첫 100자 후킹 문단]        ← 검색 미리보기 노출 영역
3) ## 📍 이 글에서 다루는 것    ← v2 신설, 목차 3~5줄 (체류시간 +)
4) ## 한 줄 요약               ← 한 문장
5) ## 시장 분석 (📊)
6) ## 투자 심리 (🧠)
7) ## 핵심 포인트 (✅)
8) ## 🔗 관련 종목 한눈에       ← v2 신설, 종목 묶음표 (동일 섹터 3~5개)
9) ## 적용할 점 (☐)            ← 체크박스 액션 3~5개
10) ## 내 생각 (💡 인사이트)    ← 한두 줄 개인 코멘트
11) [부록] 영상 정보 테이블      ← 채널/출연자/업로드일/URL
12) [부록] 투자 주의 문구
```

### 분량
- 본문 공백포함 **2,500~3,800자** 유지 (v2에서 목차·관련 종목 추가로 하한 +300).
- 한 문단 **2~4줄**(80~160자) 엄수. 5줄 넘으면 분리.
- 한 ## 섹션은 **3~5문단**.

### v2 신설 섹션 작성 룰

#### 📍 이 글에서 다루는 것 (목차)
- 굵은 글씨 3~5줄. 각 줄은 검색 키워드 1개 + 한 줄 결론.
- 모바일 네이버 앱에서 첫 스크롤에 보이도록 짧게.
- 예시:
  ```
  📍 이 글에서 다루는 것
  - **코스피 6,500선 안착** — 외국인 1.2조 매수 전환 의미
  - **SK하이닉스 18만원 회복** — D램 가격 반등 시그널 점검
  - **반도체 섹터 순환** — 다음 주도주 후보 3개
  ```

#### 🔗 관련 종목 한눈에 (종목 묶음표)
- 영상에서 언급된 종목과 같은 섹터 종목 **3~5개**.
- 표 형식: `종목명 | 시가총액 구분 | 한 줄 코멘트`.
- 검색 유입 후 평균 PV ↑가 목적 — 한 명이 여러 종목 글로 회유.
- 예시:
  ```
  | 종목 | 구분 | 코멘트 |
  |---|---|---|
  | SK하이닉스 | 대형주 | D램 사이클 직접 수혜, 18만원 안착 시 추세 회복 |
  | 삼성전자 | 대형주 | 파운드리 회복 모멘텀, SK하이닉스보다 지연 반영 |
  | 한미반도체 | 중형주 | HBM 장비, SK하이닉스 CAPEX와 동행 |
  ```
- 종목은 영상 분석 에이전트가 직접 언급한 종목 위주, 추론으로 추가 시 ⚠️ 표시.

## 5. 인포그래픽 디자인 룰 (v3 — 2026-05-13 갱신)

### 사양 (모든 PNG 공통)
- **사이즈**: **1080×1080 정사각형** (Instagram/네이버 모바일 최적). 가로형 금지 — 4/20~24 검증 결과 모바일 스크롤 피로 ↑.
- **배경**: 그라데이션 `#0F172A` → `#1E293B` (다크 네이비). 단색 `#0D1117` 대신 그라데이션 사용 — 4/27 스타일 계승.
- **상단 accent bar**: 8px, 섹션별 그라데이션 색 (market=blue/cyan, psychology=purple/pink, summary=green/cyan).
- **여백**: 외곽 padding 48px 안쪽, 콘텐츠가 1080의 95% 이상 차지하도록 빼곡하게.
- **폰트**: NanumGothic 우선, fallback Apple SD/Malgun/Noto Sans KR.

### 공통 레이아웃 (4/27 스타일 계승)
1. **상단(0~210px)**: accent bar + 날짜 배지(rounded pill) + 섹션 라벨(영문 letter-spaced) + 큰 타이틀(38px 800) + 디바이더(100×5).
2. **Hero(220~370px)**: 큰 수치/메시지 — `font-size: 100~120px`, gradient fill(`hero_from→hero_to`) + glow filter. 우측에 ▲/▼ 화살표 + 변동률.
3. **카드/포인트 영역(380~870px)**: 보조 데이터 카드 3개 가로 그리드 (각 약 313×120) + 칩 row + 포인트 1~3개.
4. **하단(880~1040px)**: 인용구 박스 (출연자 발언 한 줄) + 우측 하단 브랜딩.

### 섹션별 콘텐츠 스키마
- **market**: `stats[0]`을 hero(큰 수치, gold→red), `stats[1~3]`은 카드 3개, `chips`(5개), `points`(3개), `footer_quote/author`.
- **psychology**: `hero_msg`(45px), `traps`(3개 카드 — 각 카드에 이모지+name+desc), `correction`(큰 박스 1개), `footer_quote/author`.
- **summary**: `hero_takeaway`(26px 박스), `points`(5개 컴팩트 리스트), `footer_quote/author`.

### 텍스트 잘림 방지
- 카드 너비 313px / 폰트 40px = 한글 최대 **7~8자**. 그 이상이면 분리 라벨로.
- ❌ `stat-value: "코스피 7,999.67 — 사상 최초 8천피 턱밑"` (19자)
- ✅ `stat-value: "7,999.67"` + `stat-label: "코스피 8,000 턱밑 터치"`

### 생성 종수 (가변, 2026-05-13 갱신)
- **본문 INFOGRAPHIC_DATA에 있는 섹션만큼 자동 생성** — 강제 3종 제거. 짧은 영상 2종, 긴 영상 5종까지 가능.
- 기본 권장 3종: `market` / `psychology` / `summary`.
- 추가 가능 키 (전용 색상 토큰 ACCENTS 등록됨): `outlook`(전망), `checklist`(체크리스트), `sector`(섹터 순환), `risk`(리스크).
- **사용자 커스텀 키**도 OK — generic 빌더가 자동 fallback (회색 톤). 새 키 색상 추가는 `generate_infographics.py` ACCENTS dict에만 항목 등록.
- **insight는 이미지 만들지 않음** — 본문 최하단 `💡 *"..."*` 텍스트 한 줄.
- 데이터가 빈 dict이거나 키가 누락되면 자동 스킵.

### 렌더링
- Chrome headless `--window-size=1080,1080` 으로 HTML→PNG.
- 샌드박스에 Chrome 없으므로 Mac에서 `run_week3_full.command` (또는 유사) 더블클릭 실행.
- PNG 사이즈 정상치: 280~400KB (fallback 18KB 미만이면 Chrome 실행 실패).

이 톤북의 룰은 `08_quality_checker.py`가 자동 검사한다. 통과 기준:

- [ ] 첫 100자 후킹 문단 존재, 핵심 키워드 2회+
- [ ] **📍 이 글에서 다루는 것 목차 섹션 존재** (v2)
- [ ] **🔗 관련 종목 한눈에 표 존재, 종목 3~5개** (v2)
- [ ] 영상 정보 테이블이 첫 200자 안에 없음
- [ ] 태그 30개 이상, 롱테일 10개 이상
- [ ] **태그 첫 5개에 종목명/지수 키워드 3개 이상 포함** (v2)

- [ ] **market 인포그래픽에 종목 현재가/차트 스냅샷 카드 1개 이상** (v2)

## 11. SEO 키워드 전략 (v2 신설)

### 키워드 풀 정의
모든 글은 다음 4종 풀에서 키워드를 끌어와 배치한다:

| 풀 | 예시 | 배치 위치 |
|---|---|---|
| **지수 키워드** | 코스피, 코스닥, 코스피 8000, 5월 18일 시황 | 제목, 첫 100자, 목차 |
| **종목 키워드** | SK하이닉스, 삼성전자, LG에너지솔루션, 한화에어로스페이스 | 제목, 첫 100자, 관련 종목표, 태그 앞 5개 |
| **액션 키워드** | 매수 타이밍, 목표가, 매도 신호, 반등, 조정, 전망 | 제목 후킹, 본문 H2 |
| **롱테일 키워드** | "코스피 8000 돌파 시점", "SK하이닉스 18만원 안착 의미" | 태그 롱테일 15개 |

### 배치 룰 (강제)
- **제목**: 지수 또는 종목 키워드 1개 + 액션 키워드 1개 = 최소 2개.
- **첫 100자 후킹**: 제목과 동일한 메인 키워드 **2회 자연 반복**.
- **각 H2 섹션 첫 줄**: 해당 섹션의 메인 키워드 1회 반복.
- **태그 앞 5개**: 종목/지수 키워드 3개 이상 (검색 노출 가중치 ↑).

### 키워드 매트릭스 (제목 후보 평가용)
에이전트 4가 제목 3안 생성 시 다음 점수표로 자동 평가:

| 항목 | 점수 |
|---|---|
| 종목명 앞 12자 이내 | +3 |
| 지수 키워드 앞 12자 이내 | +2 |
| 액션 키워드 포함 | +2 |
| 숫자 포함 | +1 |
| 28~32자 | +2 |
| 브랜드 키워드 포함 | -5 |
| 검색량 낮은 키워드만 사용 | -2 |

총점 최고안 채택, 동률이면 종목 우선안.

## 12. 체류시간 강화 룰 (v2 신설)

체류시간 = 네이버 D.I.A.+ 노출 가중치 최상위 항목. 다음 4요소로 평균 1분 → 2분+ 목표.

### (1) 본문 목차 (📍)
- §3 발행 순서 3번 위치.
- 모바일 첫 스크롤에 노출 → 이탈 직전 클릭 유도.

### (2) 관련 종목 묶음표 (🔗)
- §3 발행 순서 8번 위치.
- 표 안 종목명 자체가 검색 키워드로 작용.

### (3) 인포그래픽 종목 카드 (v2 갱신)
- **market 인포그래픽 stat-card 3개 중 1개는 "오늘 영상 핵심 종목 스냅샷"**
  - 종목명 + 현재가 + 등락률 + 시가총액 또는 PER
  - 데이터 소스: 영상 분석 시 자막에서 추출, 없으면 네이버 금융 검색 (별도 fetch 도구로 보강)
- summary 인포그래픽에는 "관련 종목 3개 칩" 추가.

### (4) 본문 내 인용 박스 (선택)
- 영상의 결정적 발언 1~2개를 `>` 인용 블록으로.
- 시각 위계 분산 → 스크롤 속도 감속.

## 13. 발행 시점 분산 (v2 신설)

### 일일 발행 계획 (KST)
| 시간 | 콘텐츠 | 길이 | 스케줄 |
|---|---|---|---|
| 15:30 | **시황 요약 포스트** — 코스피·코스닥 종가, 외국인/기관 수급, 등락 상위 5개 | 1,000~1,500자 | `daily-market-summary` |
| 17:30 | **본편 영상 분석** — 12시에 만나요 영상 1편 풀 분석 | 2,500~3,800자 | `daily-stock-blog-writer` |

### 왜 분산?
- 15:30 = 장 마감 직후 "오늘 코스피", "오늘 시황" 검색 피크.
- 17:30 = 영상 업로드 직후 "12시에 만나요" 검색 피크 + 종목 분석 검색.
- 두 포스트 모두 검색 노출 면적 확보 + 서로 내부 링크 X (네이버 에디터 한계, §7 참조) 대신 첫 100자에서 자연어로 언급.

### 시황 요약 포스트 스펙
- 제목 형식: `코스피 [종가] [상승/하락] — 5월 [DD]일 외국인 [수급] [코멘트]`
- 본문: 한 줄 요약 → 지수 표 → 수급 표 → 등락 상위 5개 → 한 줄 전망 → 태그 30개
- 인포그래픽: 1장만 (market-style, 일일 지수 스냅샷)
- 발행 후 자동으로 GitHub `daily/YYYY-MM-DD-summary/` 폴더에 아카이브
